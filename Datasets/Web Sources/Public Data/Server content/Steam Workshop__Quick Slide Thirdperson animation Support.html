SCARLS686 8 Mar @ 4:12am 
Bug: Incompatibility with Anim-Based Footsteps Timing

When used with Quick Slide + ABF, all footstep sounds are lost.

Root Cause: On the first Think tick, _LastSlideState is nil vs isSliding=false, so a sliding=false net message fires on join. This makes QSlide_ResetPoseParams reset move_x/move_y to 0 every frame forever , freezing the walk animation. ABF detects footsteps via foot bone Z height — frozen animation means the threshold is never crossed.

Fix in qslide_tperson_client.lua, add one line:
if not sliding then
ply:SetPoseParameter("move_x", 0)
ply:SetPoseParameter("move_y", 0)
slidingPlayers[ply] = nil -- Add this
end
